Devuelve los templates de WhatsApp de tu company. Por defecto solo los `APPROVED`, pero podés filtrar por estado. Cada item incluye las variables esperadas extraídas del cuerpo del template, lo que te permite construir el `variables` correcto para [`POST /v1/messages/template`](/dev/endpoints/messages-template) sin tener que mantener la lista en tu lado.

## Endpoint

```
GET https://api.keebai.com/v1/templates
```

## Scope requerido

`templates:read`

## Headers

| Header | Requerido | Valor |
|---|---|---|
| `Authorization` | Sí | `Bearer kbai_pk_<token>` |

## Query params

| Param | Tipo | Default | Descripción |
|---|---|---|---|
| `page` | `number` | `0` | Página, base 0. |
| `limit` | `number` | `50` | Items por página, máximo `100`. |
| `status` | `string` | `APPROVED` | `PENDING`, `IN_REVIEW`, `APPROVED`, `REJECTED`, `PAUSED`, `DISABLED`, `IN_APPEAL`. |
| `channel_id` | `string` | — | `ObjectId` para filtrar por canal específico. Si lo omitís, devuelve templates de todos los canales de la company. |

## Ejemplo de request

<CodeGroup>

```bash curl
curl "https://api.keebai.com/v1/templates?limit=20" \
  -H "Authorization: Bearer kbai_pk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

```js JavaScript
const params = new URLSearchParams({ limit: "20" });
const resp = await fetch(`https://api.keebai.com/v1/templates?${params}`, {
  headers: { Authorization: `Bearer ${process.env.KEEBAI_API_TOKEN}` },
});
const { items } = await resp.json();
```

```python Python
import os, requests

resp = requests.get(
    "https://api.keebai.com/v1/templates",
    params={"limit": 20},
    headers={"Authorization": f"Bearer {os.environ['KEEBAI_API_TOKEN']}"},
    timeout=10,
)
resp.raise_for_status()
items = resp.json()["items"]
```

</CodeGroup>

## Respuesta

### 200 OK

```json
{
  "items": [
    {
      "id": "65f3a1b2c3d4e5f6a7b8c9d0",
      "name": "welcome_v2",
      "language": "es",
      "status": "APPROVED",
      "category": "MARKETING",
      "parameter_format": "NAMED",
      "variables": ["nombre", "monto"]
    },
    {
      "id": "65f3a1b2c3d4e5f6a7b8c9d1",
      "name": "appointment_reminder",
      "language": "es",
      "status": "APPROVED",
      "category": "UTILITY",
      "parameter_format": "NAMED",
      "variables": ["nombre", "fecha", "hora"]
    }
  ],
  "page": 0,
  "limit": 20,
  "total": 2
}
```

| Campo | Tipo | Descripción |
|---|---|---|
| `items[].id` | `string` | `ObjectId` interno del template. |
| `items[].name` | `string` | Nombre del template tal como está en Meta. Es lo que pasás en `template_name` al enviar. |
| `items[].language` | `string` | Código de idioma. |
| `items[].status` | `string` | Estado del template en Meta. |
| `items[].category` | `string` | `MARKETING`, `UTILITY`, `AUTHENTICATION`. |
| `items[].parameter_format` | `string` | `NAMED` o `POSITIONAL`. Para los nuevos integradores recomendamos solo usar `NAMED`. |
| `items[].variables` | `string[]` | Variables nombradas detectadas en el cuerpo del template. Las claves del objeto `variables` que envíes en `POST /v1/messages/template` deben coincidir con esta lista. |
| `page` | `number` | Página devuelta. |
| `limit` | `number` | Items por página. |
| `total` | `number` | Total de templates que matchean el filtro. |

### 401 / 403 / 429

Errores estándar de auth, scope y rate limit.

## Cómo usarlo en tu integración

```python
import os, requests

def fetch_template_meta(template_name: str) -> dict | None:
    resp = requests.get(
        "https://api.keebai.com/v1/templates",
        params={"limit": 100},
        headers={"Authorization": f"Bearer {os.environ['KEEBAI_API_TOKEN']}"},
        timeout=10,
    )
    resp.raise_for_status()
    for t in resp.json()["items"]:
        if t["name"] == template_name:
            return t
    return None

template = fetch_template_meta("welcome_v2")
if template is None:
    raise SystemExit("Template no aprobado todavía")

required_vars = set(template["variables"])
provided = {"nombre": "Lucio", "monto": "1500"}
if not required_vars.issubset(provided):
    raise ValueError(f"Faltan variables: {required_vars - set(provided)}")
```

<Tip>
  Cacheá la respuesta de `GET /v1/templates` por unos minutos en tu integración. Los templates nuevos no aparecen en producción de un día para otro: hay que crearlos en el portal y esperar la aprobación de Meta.
</Tip>
