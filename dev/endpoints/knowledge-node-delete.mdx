Elimina un nodo de la base de conocimientos. Si el nodo es una carpeta, todos sus descendientes (subcarpetas y documentos) se desactivan junto con ella. La eliminación es lógica (`is_active: false`); los datos siguen en la base por si el equipo de soporte necesita restaurarlos, pero dejan de aparecer en [`GET /v1/knowledge/tree`](/dev/endpoints/knowledge-tree) y de ser elegibles para [`POST /v1/knowledge/search`](/dev/endpoints/knowledge-search).

Tras eliminar, la API encola un reindexado para depurar los chunks asociados al nodo.

## Endpoint

```
DELETE https://api.keebai.com/v1/knowledge/nodes/{id}
```

## Scope requerido

`knowledge:delete`

## Headers

| Header | Requerido | Valor |
|---|---|---|
| `Authorization` | Sí | `Bearer kbai_pk_<token>` |

## Path params

| Param | Tipo | Descripción |
|---|---|---|
| `id` | `string` | `ObjectId` del nodo (carpeta o documento) a eliminar. |

## Ejemplo de request

<CodeGroup>

```bash curl
curl -X DELETE https://api.keebai.com/v1/knowledge/nodes/65f3a1b2c3d4e5f6a7b8c9d3 \
  -H "Authorization: Bearer kbai_pk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

```js JavaScript
await fetch(
  `https://api.keebai.com/v1/knowledge/nodes/65f3a1b2c3d4e5f6a7b8c9d3`,
  {
    method: "DELETE",
    headers: { Authorization: `Bearer ${process.env.KEEBAI_API_TOKEN}` },
  },
);
```

```python Python
import os, requests

resp = requests.delete(
    "https://api.keebai.com/v1/knowledge/nodes/65f3a1b2c3d4e5f6a7b8c9d3",
    headers={"Authorization": f"Bearer {os.environ['KEEBAI_API_TOKEN']}"},
    timeout=10,
)
resp.raise_for_status()
```

</CodeGroup>

## Respuesta

### 204 No Content

Sin body. La eliminación es idempotente solo dentro de la misma request: si el nodo ya estaba eliminado, devuelve `404 NOT_FOUND`.

### 401 / 403 / 404 / 429

- `403 FORBIDDEN` con `code: INSUFFICIENT_SCOPE`: el PAT no tiene `knowledge:delete`.
- `404 NOT_FOUND`: el nodo no existe, ya estaba eliminado o pertenece a otra company.

<Warning>
  Las asignaciones a workers (ver [`POST /v1/knowledge/assignments`](/dev/endpoints/knowledge-assign)) se mantienen apuntando al nodo eliminado. Como el chunk se elimina del índice en el siguiente rebuild, el asistente deja de tener acceso al contenido aunque el `knowledge_node_id` siga referenciado.
</Warning>
