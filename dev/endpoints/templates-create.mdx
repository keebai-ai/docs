Crea un template en el canal de WhatsApp indicado y lo envía a Meta para revisión. La respuesta incluye el `id` interno del template y el `status` con el que quedó. Mientras Meta no apruebe el template, no podés usarlo en [`POST /v1/messages/template`](/dev/endpoints/messages-template) — usá [`GET /v1/templates`](/dev/endpoints/templates-list) para chequear cuándo pasa a `APPROVED`.

## Endpoint

```
POST https://api.keebai.com/v1/templates
```

## Scope requerido

`templates:create`

## Headers

| Header | Requerido | Valor |
|---|---|---|
| `Authorization` | Sí | `Bearer kbai_pk_<token>` |
| `Content-Type` | Sí | `application/json` |

## Body

| Campo | Tipo | Requerido | Descripción |
|---|---|---|---|
| `channel_id` | `string` | Sí | `ObjectId` del canal WhatsApp donde se crea el template. |
| `name` | `string` | Sí | Nombre del template. Solo minúsculas, números y guiones bajos. Máx. 512. |
| `language` | `string` | Sí | Código de idioma (`es`, `es_CL`, `en_US`, etc.). Máx. 10. |
| `category` | `string` | Sí | `MARKETING`, `UTILITY` o `AUTHENTICATION`. |
| `parameter_format` | `string` | No | `NAMED` (recomendado) o `POSITIONAL`. |
| `components` | `array` | Sí | Componentes del template. Estructura idéntica a la de Meta. |
| `metadata` | `object` | No | Metadatos libres asociados al template. |

### Estructura de `components[]`

| Campo | Tipo | Descripción |
|---|---|---|
| `type` | `string` | `HEADER`, `BODY`, `FOOTER` o `BUTTONS`. |
| `format` | `string` | Solo para `HEADER`: `TEXT`, `IMAGE`, `VIDEO` o `DOCUMENT`. |
| `text` | `string` | Texto del componente. Para `BODY` admite variables `{{nombre}}`. Máx. 1024. |
| `buttons` | `array` | Solo para `type: BUTTONS`. Cada botón tiene `type` (`QUICK_REPLY`, `URL`, `PHONE_NUMBER`, `FLOW`, `VOICE_CALL`, `COPY_CODE`, `OTP`) y `text` (máx. 25). |
| `example` | `object` | Ejemplos de variables que Meta exige al revisar (`body_text`, `body_text_named_params`, `header_text`, `header_handle`). |
| `add_security_recommendation` | `boolean` | Solo para templates de autenticación. |
| `code_expiration_minutes` | `number` | Solo para templates de autenticación. |

## Respuesta

### 201 Created

```json
{
  "id": "65f3a1b2c3d4e5f6a7b8c9d2",
  "name": "welcome_v2",
  "language": "es_CL",
  "status": "PENDING",
  "category": "UTILITY",
  "parameter_format": "NAMED",
  "variables": ["nombre", "monto"]
}
```

| Campo | Tipo | Descripción |
|---|---|---|
| `id` | `string` | `ObjectId` interno del template recién creado. |
| `name` | `string` | Nombre tal como quedó registrado. |
| `language` | `string` | Código de idioma. |
| `status` | `string` | Estado inicial reportado por Meta (`PENDING` o `IN_REVIEW` apenas se crea). |
| `category` | `string` | Categoría asignada. |
| `parameter_format` | `string` | `NAMED` o `POSITIONAL`. |
| `variables` | `string[]` | Variables nombradas detectadas en el cuerpo. |

### 400 / 401 / 403 / 404 / 409 / 429

Errores estándar. Casos típicos:

- `400 BAD_REQUEST`: validación del body (nombre con caracteres inválidos, componentes mal formados, faltó `example` cuando Meta lo exige).
- `403 FORBIDDEN` con `code: INSUFFICIENT_SCOPE`: el PAT no tiene `templates:create`.
- `404 NOT_FOUND`: el `channel_id` no pertenece a tu company o no existe.
- `409 CONFLICT`: ya existe un template con el mismo `name` + `language` en ese canal.
- `502 UPSTREAM_ERROR`: Meta rechazó la creación. El campo `details` indica el motivo.

<Tip>
  Después de crear el template tenés que esperar la aprobación de Meta antes de enviarlo. Hacé polling sobre [`GET /v1/templates`](/dev/endpoints/templates-list) filtrando por `status` hasta que aparezca como `APPROVED`. La aprobación suele tardar minutos, pero puede demorar más en categorías sensibles como `MARKETING`.
</Tip>
