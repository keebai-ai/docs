Devuelve los asistentes activos de tu company. Útil principalmente para descubrir los `assistant_id` que después usás en [`POST /v1/knowledge/assignments`](/dev/endpoints/knowledge-assign). El listado solo expone metadatos básicos — para configurar prompts, herramientas o personalidad seguí usando el portal.

## Endpoint

```
GET https://api.keebai.com/v1/assistants
```

## Scope requerido

`assistants:read`

## Headers

| Header | Requerido | Valor |
|---|---|---|
| `Authorization` | Sí | `Bearer kbai_pk_<token>` |

## Query params

| Param | Tipo | Default | Descripción |
|---|---|---|---|
| `q` | `string` | — | Filtro por nombre. Búsqueda parcial, case-insensitive. |
| `page` | `number` | `1` | Página, base 1. |
| `limit` | `number` | `25` | Items por página, máximo `100`. |

## Ejemplo de request

<CodeGroup>

```bash curl
curl "https://api.keebai.com/v1/assistants?q=ventas&limit=10" \
  -H "Authorization: Bearer kbai_pk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

```js JavaScript
const params = new URLSearchParams({ q: "ventas", limit: "10" });
const resp = await fetch(`https://api.keebai.com/v1/assistants?${params}`, {
  headers: { Authorization: `Bearer ${process.env.KEEBAI_API_TOKEN}` },
});
const { items } = await resp.json();
```

```python Python
import os, requests

resp = requests.get(
    "https://api.keebai.com/v1/assistants",
    params={"q": "ventas", "limit": 10},
    headers={"Authorization": f"Bearer {os.environ['KEEBAI_API_TOKEN']}"},
    timeout=10,
)
resp.raise_for_status()
items = resp.json()["items"]
```

</CodeGroup>

## Respuesta

### 200 OK

```json
{
  "items": [
    {
      "id": "65f3a1b2c3d4e5f6a7b8c9aa",
      "name": "Ventas",
      "description": "Califica leads y agenda demos.",
      "avatar_url": "https://cdn.keebai.com/assistants/ventas.png",
      "model": "gpt-4o-mini",
      "voice_response_enabled": false,
      "created_at": "2026-03-12T10:15:00.000Z"
    }
  ],
  "page": 1,
  "limit": 10,
  "total": 1
}
```

| Campo | Tipo | Descripción |
|---|---|---|
| `items[].id` | `string` | `ObjectId` del asistente. Es lo que usás como `assistant_id` en otros endpoints. |
| `items[].name` | `string` | Nombre. |
| `items[].description` | `string` | Descripción corta visible en el portal. |
| `items[].avatar_url` | `string` | URL pública del avatar, si fue cargado. |
| `items[].model` | `string` | Modelo configurado (ej. `gpt-4o-mini`). |
| `items[].voice_response_enabled` | `boolean` | Indica si las respuestas de voz están habilitadas. |
| `page` / `limit` / `total` | `number` | Paginación estándar. |

### 401 / 403 / 429

- `403 FORBIDDEN` con `code: INSUFFICIENT_SCOPE`: el PAT no tiene `assistants:read`.
