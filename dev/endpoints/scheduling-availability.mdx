Calcula los slots libres para un servicio en una fecha puntual. El motor considera horarios de los profesionales, citas ya agendadas, buffers del servicio (`bufferBefore`, `bufferAfter`) y restricciones horarias (`time_rules.earliest_start`, `time_rules.latest_end`). Si no se especifica `branchId`, se usa la primera sucursal del tenant.

## Endpoint

```
GET https://api.keebai.com/v1/scheduling/availability
```

## Scope requerido

`scheduling:availability:read`

## Query params

| Param | Tipo | Requerido | Descripción |
|---|---|---|---|
| `serviceId` | `string` | Sí | Servicio a agendar. Define duración y restricciones del slot. |
| `date` | `string` | Sí | Fecha en formato `YYYY-MM-DD`. Si es anterior a hoy en el tz de la sucursal, devuelve `results: []`. |
| `branchId` | `string` | No | Sucursal. Si se omite, se usa la primera del tenant. |
| `professionalId` | `string` | No | Restringir el cálculo a un profesional específico. |

## Respuesta

### 200 OK

```json
{
  "results": [
    {
      "id": "66c1d2e3f4a5b6c7d8e9f0a1",
      "name": "María",
      "last_name": "Soto",
      "timeSlots": [
        { "time": "09:00" },
        { "time": "09:30" },
        { "time": "10:00" }
      ]
    }
  ]
}
```

| Campo | Tipo | Descripción |
|---|---|---|
| `results[].id` | `string` | Id del profesional. |
| `results[].timeSlots[].time` | `string` | Hora de inicio del slot (`HH:mm`) en el tz de la sucursal. |

### 400 Bad Request

`date` con formato inválido, `serviceId` ausente, sucursal del tenant inexistente.

### 401 Unauthorized · 403 Forbidden

Mismas semánticas que el resto de la API pública.

## Patrones comunes

### UI de selección "fecha → profesional → hora"

1. El usuario elige una fecha.
2. Tu app llama a `GET /availability?date=<fecha>&serviceId=...&branchId=...`.
3. La respuesta agrupa los slots por profesional, listo para pintar como columnas o filas.
